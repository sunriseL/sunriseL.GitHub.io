Trapping Rain Water

2019-03-03

#leetcode

原题

https://leetcode.com/problems/trapping-rain-water/

题目描述

Given n non-negative integers representing an elevation map where the width of each bar is 1, compute how much water it is able to trap after raining.

例子

Example 1:

Input: [0,1,0,2,1,0,1,3,2,1,2,1]
Output: 6

解法一

总结

这道题的解法是自己想出来的，代码非常的不优雅，考虑看一下标准解法吸收一下
基本思路是通过找到一个最高点，然后继续往后搜索，直到搜索到比之前的最高点持平或者更高的的点，将这两个中间的部分求和，同时把中间的实的部分减掉，实的部分在搜索的过程就求和了，因此是一次扫描过程，如果是这样的话会出现在最后一个高点后的小水坑无法被记录，因此在搜索最高点的过程中会记录次高点，如果在上一个最高点的后面高度不是单调递减的话，就从最高点开始再搜索一遍，同时高度限制在次高点的高度。如此往复，直到只有单调递减的点为止。
